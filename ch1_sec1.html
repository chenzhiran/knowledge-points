1.1 计算机程序与编程概述

Introduction to Computer Programs and Programming

1.1.1 什么是程序（What is a Program）

1.1.1.1 程序的定义：一组指令的有序集合 A program is an ordered collection of instructions

图1：程序就像一份行动清单，英雄按照清单逐条执行

想象一下，安雅（Anya）站在 Kithgard 地牢的入口，前方是一条布满尖刺的走廊。她很聪明，但她不会自己行动——她需要你告诉她怎么走。

程序（Program），就是你写给计算机（或英雄）的一份"行动清单"。这份清单里的每一行，都是一条明确的指令，按照从上到下的顺序排列好。计算机会忠实地按照这个顺序，一条一条地执行。

就好比你给朋友写一张纸条：

hero.moveRight()
hero.moveDown()
hero.moveRight()

这三行代码就是一个完整的程序。它简单、明确、有序——这就是程序最本质的样子。

一句话记忆：程序 = 一组按顺序排列的指令集合。
One-liner: A program is an ordered collection of instructions.

1.1.1.2 程序的执行方式：逐行顺序执行 Programs execute line by line in sequence

图2：程序像读书一样从第一行读到最后一行

计算机执行程序时，就像读一本书一样——从第一行开始，一行一行往下读，读完一行、做完一行，再去读下一行。

当安雅收到上面那三行指令时：

  • ● 她先执行第 1 行：向右走一步 ✓
  • ● 第 1 行做完后，执行第 2 行：向下走一步 ✓
  • ● 第 2 行做完后，执行第 3 行：向右走一步，抓住宝石！✓

她绝不会跳过某一行，也不会先做第 3 行再做第 1 行。这种"从头到尾、逐行执行"的方式，就是顺序执行（Sequential Execution）。

小提示：如果你把第 2 行和第 3 行的顺序调换，安雅走的路就完全不一样了——可能还会撞上尖刺！顺序很重要。

1.1.1.3 程序与计算机的关系 Relationship between programs and computers

图3：没有程序的计算机就像没有乐谱的钢琴

计算机本身其实是个"听话但不会思考"的家伙。它跑得飞快、算得精准，但有一个问题：它不知道自己该做什么。

程序就是告诉计算机"该做什么"的东西。没有程序的计算机，就像没有乐谱的钢琴——钢琴再好，也发不出曲子。

在 CodeCombat 的世界里：

  • ● 计算机 = 游戏引擎（它能让英雄走动、攻击、拾取物品）
  • ● 程序 = 你写的代码（它告诉游戏引擎让英雄做什么）
  • ● 英雄（hero） = 执行你指令的对象

你就是那个写乐谱的作曲家！

1.1.1.4 源代码与可执行代码 Source code vs. executable code

图4：源代码经过解释器翻译后，计算机才能理解

你在编辑器里写的那些文字——hero.moveRight() 之类的——叫做源代码（Source Code）。源代码是给人看的，用人类能读懂的编程语言写成。

但计算机的"母语"其实是由 0 和 1 组成的机器语言。所以在你点击"运行"按钮后，背后会有一个翻译官把你的 Python 代码翻译成计算机能理解的指令——这个翻译后的东西就是可执行代码（Executable Code）。

不过别担心！在 CodeCombat 和大多数 Python 开发中，这个翻译过程是全自动的。你只需要专注于写好源代码，剩下的交给 Python 解释器就好。

概念类比说明
源代码 Source Code你写的中文菜谱人类可读的指令
可执行代码 Executable Code厨师实际的操作计算机能理解的指令
解释器 Interpreter翻译官把源代码翻译给计算机

1.1.2 什么是编程（What is Programming）

1.1.2.1 编程的定义：编写计算机指令的过程 The process of writing instructions for a computer

图5：编程的四步循环——想、写、测、改

编程（Programming）就是把你脑子里的想法——"我要让安雅走到宝石那里"——翻译成计算机能执行的代码的过程。

编程的核心步骤是：

  • 1️⃣ 想清楚：我要让英雄做什么？（向右走，然后向下走）
  • 2️⃣ 写出来：用编程语言表达（hero.moveRight()）
  • 3️⃣ 测试它：点击运行，看看英雄是否按你的想法行动
  • 4️⃣ 修正它：如果英雄走错了路，修改代码再试一次

恭喜！当你在 Kithgard 地牢的第 1 关写下第一行 hero.moveRight() 并点击运行时，你就已经在编程了！

1.1.2.2 编程语言的作用 The role of programming languages

图6：编程语言是人类与计算机之间的桥梁

人类说中文、英文、法文……计算机只懂 0 和 1。编程语言就是人类和计算机之间的"桥梁语言"。

编程语言有很多种（就像人类语言有很多种一样），比如 Python、JavaScript、Java、C++ 等等。它们各有特色，但都是为了同一个目的：让人类能方便地写出计算机能执行的指令。

在 CodeCombat 中，你使用的是 Python 语言。

1.1.2.3 Python 作为编程语言的特点 Python as a programming language

图7：Python——初学者最友好的编程语言之一

为什么我们选择 Python？因为它是对初学者最友好的编程语言之一：

  • ● 语法简洁：代码看起来很接近英文，读起来不费劲
  • ● 不需要分号：很多语言每行末尾都要加 ;，Python 不用
  • ● 用缩进表示结构：代码的层次感一目了然（后面会详细学）
  • ● 全球最流行之一：从网站开发到人工智能，到处都用 Python
  • ● 庞大的社区：遇到问题随时能找到帮助

看看这段 Python 代码有多自然：

hero.moveRight()
hero.moveDown()
hero.moveRight()

即使你从没学过编程，也大概能猜到它的意思——让英雄向右走、向下走、再向右走。

1.1.2.4 编程与问题解决 Programming and problem solving

图8：编程的核心是解决问题——理解、设计、编写、验证

编程不仅仅是"打字"——它的核心是解决问题（Problem Solving）。

在第 3 关"幽影守卫"中，安雅不能直接冲过去，因为食人魔矮人挡在路上。你需要想出一条绕开它的路线，然后用代码实现这个计划。

这个过程就是：

  • 1️⃣ 理解问题：食人魔在中间，我不能被它看到
  • 2️⃣ 设计方案：我可以从雕像后面绕过去
  • 3️⃣ 编写代码：把方案翻译成一系列移动指令
  • 4️⃣ 验证结果：运行代码看看安雅是否安全通过

编程教会你的不只是 Python 语法，更是一种系统化思考和解决问题的能力。这种能力在任何领域都有用！

1.2 对象与方法

Objects and Methods

1.2.1 对象的概念（The Concept of Objects）

1.2.1.1 对象的定义：具有属性和行为的实体 An entity with attributes and behaviors

图13：对象 = 属性（特征）+ 方法（本领）

在编程世界里，对象（Object）是一个核心概念。简单来说，对象就是一个"东西"——一个拥有特征和本领的东西。

在 CodeCombat 中，hero（英雄）就是一个对象。让我们看看塔林（Tharin）有什么：

类别举例
属性（特征）生命值、速度、位置
方法（本领）向右走、向左走、向上走、向下走

对象 = 属性 + 方法，就是这么简单！

1.2.1.2 对象的类比：现实世界中的事物 Real-world analogy of objects

图14：猫、手机、英雄——都是对象！

其实你身边到处都是"对象"！来看几个例子：

一只猫

  • ● 属性：名字是"咪咪"、颜色是橘色、体重 5kg

  • ● 属性：品牌是 Apple、颜色是黑色、电量 80%

游戏中的英雄塔林

看出规律了吗？任何有特征、会做事的东西，在编程中都可以被看作一个"对象"。

1.2.1.3 Python 中一切皆对象 Everything in Python is an object

图15：在 Python 中，数字、字符串、英雄……全都是对象

Python 有一个很酷的设计哲学：一切皆对象（Everything is an object）。

不仅仅是英雄是对象——数字 3 是对象、字符串 "Hello" 是对象、甚至你写的函数也是对象。

不过现在你只需要记住一个对象就好：hero——你在 CodeCombat 中的英雄。其他的以后慢慢认识！

1.2.1.4 对象的标识符/名称 Object identifier/name，如 hero

图16：无论选择安雅还是塔林，在代码中都叫 hero

每个对象都需要一个名字（标识符 / Identifier），这样我们才能在代码中引用它。

在 CodeCombat 中，你的英雄的名字就是 hero。无论你选择安雅还是塔林，在代码中都用 hero 来称呼他们。

hero.moveRight()   # "hero" 就是对象的名字

就像你在教室里喊"塔林，向右走！"一样，在代码里你写 hero.moveRight() 就是在对英雄下达指令。

注意：hero 这个名字是 CodeCombat 预先设定好的，不能随便改。

1.2.1.5 对象的属性 Object attributes/properties

图17：属性描述对象"是什么样的"

属性（Attributes / Properties）描述了对象"是什么样的"——它的特征、状态、数据。

英雄的属性例如：当前生命值、当前位置（在地图上的哪个格子）、移动速度。

虽然在前 6 关中我们不需要直接读取属性值，但了解它们的存在很重要：属性是对象携带的信息。

1.2.1.6 对象的行为/动作 Object behaviors/actions

图18：方法是对象能执行的动作

行为（Behaviors）描述了对象"会做什么"——它能执行的动作。在编程中，我们把对象的行为叫做方法（Method）。

在第 1–6 关中，英雄能做的动作是四个方向的移动：moveRight()、moveLeft()、moveUp()、moveDown()。

1.2.2 方法的概念（The Concept of Methods）

1.2.2.1 方法的定义：对象能执行的动作 Actions that an object can perform

图19：对象是"谁"，方法是"做什么"

方法（Method）是附属于对象的一段可执行代码，代表对象能够执行的一个动作。

当你写 hero.moveRight() 时：

  • ● hero 是谁？→ 英雄（对象）
  • ● moveRight() 做什么？→ 向右移动（方法）

翻译成人话就是："嘿英雄，请向右移动！"

1.2.2.2 方法与函数的关系 Relationship between methods and functions

图20：方法是属于对象的函数

你可能会在学习中听到两个词：方法（Method）和函数（Function）。它们非常相似，核心区别是：

  • ● Function（函数）：独立存在的一段可执行代码
  • ● Method（方法）：属于某个对象的函数

类比：

  • ● "跑步"是一个通用动作（函数）
  • ● "塔林跑步"是塔林的专属动作（方法）

在 CodeCombat 中，moveRight() 是 hero 的方法——只有英雄能做这个动作。

1.2.2.3 方法的调用/执行 Calling/invoking a method

图21：调用方法就像呼唤英雄去做某件事

写下一个方法并让它执行，叫做调用方法（Calling a method）。

hero.moveRight()  # 调用 moveRight 方法

当 Python 读到这一行时，它会：① 找到 hero 这个对象 → ② 在 hero 身上找到 moveRight 这个方法 → ③ 执行这个方法（让英雄向右移动）。

1.2.2.4 方法调用的语法：圆括号 () Syntax of method calls: parentheses

图22：圆括号的意思是"现在就做！"

注意到方法名后面那对圆括号 () 了吗？它非常重要！

hero.moveRight()   # ✓ 正确：有括号，调用方法
hero.moveRight     # ✗ 错误：没括号，不执行

圆括号的意思是："现在就做！"。把它想象成按门铃——moveRight 是门铃按钮本身，moveRight() 是按下门铃的动作。

牢记：没有那对括号，方法就不会执行。这是初学者常见的坑！

1.2.2.5 方法的命名规范 Method naming conventions

图23：好的方法名一看就知道它做什么

在 CodeCombat 中，方法名使用驼峰命名法（camelCase）：第一个单词小写，后面每个单词首字母大写。

方法名命名分析
moveRightmove（移动）+ Right（右边）→ 向右移动
moveLeftmove（移动）+ Left（左边）→ 向左移动
moveUpmove（移动）+ Up（上面）→ 向上移动
moveDownmove（移动）+ Down（下面）→ 向下移动

好的方法名本身就能告诉你它做什么——代码像读英语一样自然。

图24：蛇形命名法与驼峰命名法对比

小知识：Python 官方推荐函数和方法使用 snake_case（蛇形命名法，如 move_right），但 CodeCombat 使用了 camelCase（驼峰命名法，如 moveRight）。两种风格都很常见，关键是在同一个项目中保持一致。

1.2.2.6 内置方法与自定义方法 Built-in methods vs. custom methods

图25：内置方法是游戏已经写好的，直接用就行

在 CodeCombat 中，四个移动方法都是内置方法（Built-in Methods）——游戏已经帮你写好了，你直接用就行。在未来的学习中，你还会学到如何自己创建方法。

1.2.4 常用移动方法（Common Movement Methods）

1.4.3.5 浮点数的精度问题简介 Introduction to floating-point precision

💡 为什么 0.1 + 0.2 ≠ 0.3？

计算机使用二进制（0 和 1）来存储所有数据，包括小数。但就像十进制无法精确表示 1/3（只能写 0.333...），二进制也无法精确表示某些十进制小数（如 0.1）。这导致浮点运算会产生极其微小的误差。这不是 Python 的 bug，所有编程语言都有这个问题。好消息是：在 CodeCombat 的前 6 关中，你主要使用整数参数，完全不会遇到这个问题！

print(0.1 + 0.2)   # 输出: 0.30000000000000004

这不是 bug，是计算机用二进制存储小数时的微小误差。前 6 关中通常用整数参数，不太会遇到这个问题。

负数是小于零的数，写法是在正数前面加一个减号 -。在数学中，负数表示与正数相反的方向或概念：温度 -5°C 表示零下五度，海拔 -100m 表示在海平面以下 100 米。在编程中，负数同样表示"反向"。当你对 moveDown 传入 -1 时，它的效果就是"反方向"——即向上移动 1 步。

1.6 顺序执行

Sequential Execution

1.6.1 顺序结构的定义（Definition of Sequential Structure）

1.6.1.1 代码按书写顺序从上到下执行Code executes from top to bottom

图53：顺序结构——代码从第一行到最后一行，绝不跳行

顺序结构（Sequential Structure）就是代码从第一行开始，按照书写顺序，一行一行向下执行。前 6 关的所有解法都是纯顺序结构。

1.6.1.2 每条语句执行一次Each statement executes once

💡 "一次且仅一次"——顺序结构的特征

在纯顺序结构中，每一行代码只会被执行恰好一次——不会多也不会少。这意味着如果你写了 5 行代码，程序运行时就会精确地执行 5 个动作。想让英雄做 10 个动作？那你就得写 10 行代码（或者用参数优化，但总的移动"次数"不变）。后面学到循环结构后，你就能让一段代码重复执行多次——那就不再是纯顺序结构了。

1.6.1.3 前一条语句完成后才执行下一条Next statement executes after previous completes

💡 "排队执行"——不会插队

顺序执行意味着代码是"排队"的：第 1 行必须完全执行完毕后，第 2 行才会开始执行。在 CodeCombat 中，这表现为英雄必须走完一步后才会走下一步——你不会看到英雄同时做两件事。这种"阻塞式"执行方式让程序行为完全可预测：你可以准确地知道在任意时刻英雄在哪里、在做什么。这个特性对于时机策略非常重要，如下面第 3b 关的例子所示。

在第 3b 关"狭路相逢"中，严格的顺序执行让时机策略成为可能：

hero.moveRight()    # 时刻1：走到门廊处
hero.moveLeft()     # 时刻2：退回来！
hero.moveRight(2)   # 时刻3：趁机冲过去！

1.6.1.4 顺序结构是最基本的控制结构Sequential structure is the most basic control structure

💡 三大控制结构——程序的"骨架"

所有计算机程序，无论多复杂，都是由三种基本控制结构组合而成的：顺序（从上到下依次执行）、选择（根据条件选择不同路径，如 if/else）、迭代（循环重复执行某段代码，如 while/for）。前 6 关只用到了顺序结构——这是最基础的，也是另外两种结构的"地基"。后面的关卡会逐步引入循环（Unit 3）和条件判断，让你写出更强大的程序。

所有程序都由三种控制结构组成：顺序（Sequence）、选择（Selection）、迭代（Iteration）。前 6 关只用到了最简单的顺序结构。

1.6.2 顺序结构的特点（Characteristics of Sequential Structure）

1.6.2.1 确定性：执行顺序固定Deterministic: fixed execution order

💡 确定性——每次运行结果都一样

顺序结构的第一个重要特点是确定性：代码的执行顺序是固定的，不会随机改变。这意味着同样的代码每次运行都会产生完全相同的结果。在 CodeCombat 中，只要你不修改代码，英雄每次走的路线都一模一样。这个特性让程序变得可预测、可调试——如果代码有错，错误也是稳定复现的，方便你找到并修复它。

1.6.2.2 可预测性：结果可预知Predictable: results are foreseeable

💡 可预测性——你可以"脑中运行"代码

因为顺序结构是确定性的，你可以在脑子里"模拟运行"代码——逐行阅读，想象英雄的每一步，就能预知最终结果。这是一个非常有用的能力，叫做"代码追踪"（Code Tracing）或"干运行"（Dry Run）。在点击运行按钮之前，先用脑子走一遍代码，可以帮你提前发现错误。试试在纸上画出地图，然后用铅笔跟着代码一步步画出英雄的路线。

1.6.2.3 语句顺序影响结果Order of statements affects outcome

💡 顺序不同，结果天差地别！

想象你做饭的步骤：先切菜再炒菜（正确！），如果改成先炒菜再切菜（怎么炒一整棵白菜？）。编程也是一样——调换语句的顺序可能导致完全不同的结果。比如在第 3 关中，安雅需要先向右走到雕像旁，再向上走躲起来。如果你把两行调换，她会先向上撞墙，然后向右暴露在食人魔面前——结果完全不同！

来看第 3 关"幽影守卫"。如果调换前两行的顺序，安雅可能撞墙或被食人魔发现！

记住：写代码就像给人导航——"先左转再直行"和"先直行再左转"可能带你去完全不同的地方。

1.6.3 多行代码的组织（Organizing Multiple Lines of Code）

1.6.3.1 一行一条语句One statement per line

💡 一行一指令——Python 的基本格式

Python 的基本规则之一：每行写一条语句。不要把多条指令挤在同一行。虽然 Python 在技术上允许用分号 ; 把多条语句写在一行（如 hero.moveRight(); hero.moveDown()），但这样做会让代码难以阅读。就像写作文要分段一样，每行一条语句让代码清晰、整洁、容易理解。

1.6.3.2 语句的分解与组合Decomposing and combining statements

💡 分解与组合——灵活运用参数

同一个任务可以用不同的语句组合来完成。"向右走 3 步"既可以分解为 3 行 hero.moveRight()，也可以组合成 1 行 hero.moveRight(3)——效果相同。学会灵活地分解（让每步清晰）和组合（让代码简洁）语句，是编程的基本功。通常的原则是：在不牺牲可读性的前提下，尽量用参数来减少重复的行。

1.6.3.3 代码的逻辑分组Logical grouping of code

💡 逻辑分组——给代码"分段落"

当代码行数较多时（比如第 4 关有 12 行），可以按照逻辑意义将代码分组。例如："第一段：从入口走到拐角"、"第二段：从拐角走到宝石"。你可以用空行或注释来分隔这些逻辑组，让代码读起来像一篇结构清晰的短文。

1.6.3.4 空行的使用提高可读性Using blank lines for readability

💡 空行——代码中的"呼吸空间"

空行不会被 Python 执行（它什么都不做），但对人类阅读代码帮助巨大。在不同的逻辑段落之间加一个空行，就像文章中的段落间距一样，让读者能轻松分辨代码的不同部分。好习惯从现在开始培养！

空行不会被执行，但能让人类读代码时"喘口气"。就像写文章要分段一样，写代码也要适当留白！

💡 大小写敏感——一个字母大小写不同就报错！

Python 是大小写敏感（Case Sensitive）的语言，这意味着 hero、Hero、HERO 是三个完全不同的名字！同样地，moveRight 和 MoveRight 和 moveright 也是完全不同的。这是初学者最常犯的错误之一。记住 CodeCombat 中的规则：对象名 hero 全小写，方法名第一个单词小写、后面每个单词首字母大写（驼峰命名法）。

hero.moveRight()    # ✓ 正确
Hero.moveRight()    # ✗ 错误！"Hero" ≠ "hero"
hero.MoveRight()    # ✗ 错误！"MoveRight" ≠ "moveRight"
hero.moveright()    # ✗ 错误！"moveright" ≠ "moveRight"
记忆技巧：hero 全小写，move 小写，方向首字母大写（Right、Left、Up、Down）。

1.9.2 Python 基本规则（Basic Python Rules）

1.9.2.1 每行一条语句One statement per line

💡 一行一语句——Python 的书写规范

Python 使用换行符来分隔不同的语句。这意味着每写完一条指令就按一次回车，开始新的一行。这个规则让 Python 代码看起来干净整洁，每行只做一件事，阅读时一目了然。如果你在其他语言（如 JavaScript）中见过 ; 结尾的语句，在 Python 中不需要——换行本身就标志着一条语句的结束。

1.9.2.2 缩进的重要性预告Preview: importance of indentation

💡 缩进预告——Unit 3 的重要准备

在前 6 关中，你的所有代码都是顶格写的（没有缩进）。但在后面学到循环（while True:）和条件判断（if）时，你就需要用缩进（通常是 4 个空格）来表示"哪些代码属于循环/条件内部"。Python 用缩进来标识代码块的层级关系——这是 Python 的独特设计。现在先记住：缩进在 Python 中不只是美观，它有语法意义！

1.9.2.3 Python 关键字Python keywords，初步了解

💡 关键字——Python "保留"的特殊单词

Python 有一些被"保留"的单词，叫做关键字（Keywords），如 while、True、False、if、else、for、def、return 等。这些词在 Python 中有特殊含义，你不能用它们作为变量名或自定义标识符。在前 6 关中你还不会直接使用这些关键字，但了解它们的存在可以避免将来踩坑——比如你不能写 True = 3 这样的代码。

1.9.2.4 标识符命名规则Identifier naming rules

💡 标识符命名——给"东西"取名的规则

在 Python 中，标识符（变量名、函数名、方法名等）需要遵循这些规则：① 只能包含字母、数字和下划线 _；② 不能以数字开头（1hero 不行，hero1 可以）；③ 不能使用 Python 关键字；④ 区分大小写（Hero 和 hero 是不同的名字）。好的命名习惯：名字应该有意义（moveRight 比 mr 好）、保持一致的风格（全用 camelCase 或全用 snake_case）。在 CodeCombat 中，hero 已经帮你取好了名字，但在将来的编程中你需要自己给变量取名。
